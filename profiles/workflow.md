# Before You Write Code

- Read the entire task description twice
- Identify all explicit requirements and edge cases
- Plan your approach before typing

# While Writing Code

- Handle every edge case mentioned in the requirements
- Write a docstring for each function explaining what it does
- If you extract helper functions, give them clear descriptive names
- Validate function inputs at the start of each public function

# After Writing Code

- Verify your solution addresses every requirement in the task
- Remove any unused imports or dead code
- Ensure your code would pass a linter without warnings
