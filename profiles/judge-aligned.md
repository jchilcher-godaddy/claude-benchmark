# Implementation Standards

## Readability
- Use descriptive variable and function names that reveal intent
- Structure code in logical sections with consistent formatting

## Architecture
- Separate concerns into focused functions with single responsibilities
- Use appropriate abstractions — not too many, not too few

## Requirements
- Implement every stated requirement completely
- Respect all constraints and specifications exactly as written

## Correctness
- Handle all edge cases mentioned or implied by the requirements
- Validate inputs and raise clear errors for invalid arguments
