# Avoid These Mistakes

- Do not add features or behaviors beyond what the task specifies
- Do not leave dead code, commented-out code, or unused imports
- Do not ignore edge cases mentioned in the requirements
- Do not use overly clever one-liners when a clear multi-line version is more readable
- Do not create deeply nested code when guard clauses would work
- Do not omit error handling for explicitly stated error conditions
- Do not write functions longer than 30 lines without extracting helpers
